NAME

geopm_error - geopm error handling  

SYNOPSIS

#include <geopm_error.h>

Link with -lgeopm (MPI) or -lgeopmpolicy (non-MPI)

void geopm_error_message(int err,
                         char *msg,
                         size_t size);
 

DESCRIPTION

All geopm(3) C APIs which can have an error condition occur during runtime return an error number. This is the way geopm(3) handles errors, and almost all of the interfaces behave this way. This man page describes how to interpret these returned error codes. A returned error number of zero by a geopm API indicates success. If the error number returned by a geopm API is positive then this indicates a generic system error, and if the error number is negative this indicates a geopm(3) specific error has occurred. Any non-zero error number can be converted into a descriptive string with the geopm_error_message() function. The GEOPM specific error numbers are enumerated in the geopm_error.h header file and they are described below. The system error numbers are documented in the errno(3) man page.

Runtime error
GEOPM_ERROR_LOGIC = -2
Logic error
GEOPM_ERROR_INVALID = -3
Invalid argument

The geopm_policy_c pointer is NULL, use geopm_policy_create
GEOPM_ERROR_FILE_PARSE = -5
Unable to parse input file
GEOPM_ERROR_LEVEL_RANGE = -6
Control hierarchy level is out of range
GEOPM_ERROR_CTL_COMM = -7
Communication error in control hierarchy
GEOPM_ERROR_SAMPLE_INCOMPLETE = -8
All children have not sent all samples
GEOPM_ERROR_POLICY_UNKNOWN = -9
No policy has been set
GEOPM_ERROR_NOT_IMPLEMENTED = -10
Feature not yet implemented
GEOPM_ERROR_NOT_TESTED = -11
Feature not yet tested
GEOPM_ERROR_PLATFORM_UNSUPPORTED = -12
Current platform not supported or unrecognized
GEOPM_ERROR_MSR_OPEN = -13
Could not open MSR device
GEOPM_ERROR_MSR_READ = -14
Could not read from MSR device
GEOPM_ERROR_MSR_WRITE = -15
Could not write to MSR device
GEOPM_ERROR_OPENMP_UNSUPPORTED = -16
Not compiled with support for OpenMP
